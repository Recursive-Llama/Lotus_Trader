## **7.2 Math Learning System**

The math layer evaluates behaviour within each scope using a composite field.

At a high level, edge is not a single number but a composite field:

edge=ŒîRR√óreliability√ó(support+magnitude+time+stability)√ódecay

Each component contributes:

* **\(\Delta RR\)** ‚Äî measures improvement over a global baseline (multiplicative)
* **reliability** ‚Äî captures variance and consistency (multiplicative)
* **integral** ‚Äî combines four additive factors:
  * **support** ‚Äî encodes how many times a pattern has been observed
  * **magnitude** ‚Äî measures typical reward
  * **time** ‚Äî tracks confirmation speed
  * **stability** ‚Äî tracks how edge behaves over time
* **decay** ‚Äî penalises patterns whose performance is degrading (multiplicative)

This multi-dimensional approach prevents Lotus from being fooled by high-variance patterns or patterns that worked once but are decaying.

Only when this field is strong and stable within a scope does Lotus promote a pattern into a lesson.

Each scope produces **lessons** ‚Äî condensed statistical truths that shape allocation, timing, conviction, and behaviour.

Lessons do not stay abstract.
When a pattern repeatedly produces edge inside a scope, Lotus distils that lesson into behavioural adjustments: sizing multipliers, stricter or looser thresholds, and timing preferences.
This creates a complete loop: behaviour ‚Üí outcome ‚Üí lesson ‚Üí behaviour.

Math governs what stays, what adapts, and what is abandoned.
It is the ultimate filter.

If math disagrees, math wins.
